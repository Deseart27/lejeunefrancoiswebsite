Terms and conditions


Project carried out as part of the validation from the Data Science bootcamp of Le wagon 🚃

Terms and conditions is a chrome extension that intelligently detects the terms and conditions of a website and summarizes it in a few lines thanks to a dedicated Machine learning model.

Some technical details

Django

Thanks to its REST framework, Django allowed us to create a flexible API calling an ML model and a smart scrapper.

Machine Learning

Terms and conditions retrieved are cleaned and summarized via Sumy, a library based on NLTK.

Deployment on Heroku & Gcloud

For performance and maintainability reasons, we split the deployment on two hosts: Heroku (scrapper) and Gcloud (ML model).

Selenium

Thanks to its "ChromeDriver" WebDriver, Selenium allowed us to create a smart scrapper which detects the terms and conditions of a website and extracts them automatically.

My role in the project

Back-end architecture

Realization of the architecture of the back-end, it's to say : to organize the communication between our extension (JS), the smart scrapper (Python), our database (GoogleBigQuery) and our Machine Learning model.

Creation of APIs

Creation of two dedicated APIs with Django REST framework. Each API get specific endpoints : smart detection of the terms and condition (accuracy : 85%), extraction, verification in GBQ, text cleaning, ML model calling...

Deployment of the solution

Scrapper deployment on Heroku with Django for Heroku + gunicorn, and ML model deployment via Gcloud Environnement flexible App Engine.

Preparation of the presentation

Organisation and design of the slides of the presentation with Canva + Photoshop.
